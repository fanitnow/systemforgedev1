Data Capture & Storage Setup

This guide walks you through connecting your platform to a real database so every lead, audit, client, and dollar is captured and stored permanently. Takes about 45 minutes. Do it once, runs forever.

Overview

What gets captured

Every touchpoint on your platform generates data. Here's exactly what gets saved and why each piece matters to your business.

Data TypeWhat It ContainsWhy It Matters
audit_leadsBusiness type, industry, audit conversation, revenue leak amount, missing systems, timestamp, IPEvery person who runs an audit — even if they don't buy. This is your warm lead list.
audit_reportsFull audit results: leak amount, missing systems, recommended fixes, business name if providedThe actual output of every audit. Searchable. Referenceable when you follow up.
checkout_submissionsName, email, phone, business name, industry, revenue range, pain points, tools, goals, tier selected, amountEvery person who fills out checkout — paid or not. Your active sales pipeline.
paid_clientsAll checkout data + payment confirmed flag, Stripe payment ID, tier, delivery status, delivery dateYour paying client database. Tracks build progress, delivery, and follow-up schedule.
page_viewsPage visited, referrer source, timestamp, session IDWhere traffic comes from. Tells you which outreach channel is working.
modal_clicksWhich CTA button was clicked, what tier, timestampConversion funnel data — see where people drop off before buying.
Your most valuable asset: The audit_leads table. Every person who runs your free audit is a warm lead — they already know they have a problem. This list is worth more than anything else you'll build. Guard it, follow up on it weekly.
Step 01

Create your Supabase account

Supabase is your database. Free plan holds up to 500MB — enough for tens of thousands of leads and clients. Takes 5 minutes to set up.

1

Go to supabase.com

Click "Start your project" — sign up with your Google account or email. It's free.

2

Create a new project

Click "New Project." Name it systemforge-ai. Choose a strong database password and save it somewhere safe (1Password or your notes app). Select region: US East.

⚠️ Save your database password. You cannot recover it. Store it in your phone notes right now.
3

Wait for project to launch

Takes about 60 seconds. You'll see a green "Project is ready" status. Then move to Step 2.

Step 02

Create your database tables

In Supabase, go to the SQL Editor (left sidebar). Click "New query." Paste each block below and click "Run." Do them in order.

How to find SQL Editor: Inside your Supabase project → left sidebar → click the database icon → "SQL Editor" → "New query"

Table 1 — Audit Leads

Copy SQL -- Captures everyone who runs the free audit tool CREATE TABLE audit_leads ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, created_at TIMESTAMPTZ DEFAULT now(), session_id TEXT, business_type TEXT, industry TEXT, messages JSONB, -- full conversation stored as JSON leak_amount TEXT, missing_systems TEXT[], recommended_fixes TEXT[], report_shown BOOLEAN DEFAULT false, cta_clicked BOOLEAN DEFAULT false, source_page TEXT, ip_address TEXT );

Table 2 — Checkout Submissions

Copy SQL -- Captures everyone who fills out the checkout form CREATE TABLE checkout_submissions ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, created_at TIMESTAMPTZ DEFAULT now(), first_name TEXT, last_name TEXT, email TEXT, phone TEXT, business_name TEXT, industry TEXT, location TEXT, revenue_range TEXT, website TEXT, pain_points TEXT[], pain_desc TEXT, tools_used TEXT[], goal_desc TEXT, tier_selected INTEGER, tier_name TEXT, amount INTEGER, -- in cents: 9700, 75000, 250000 payment_status TEXT DEFAULT 'pending', stripe_payment_id TEXT, consented_to_terms BOOLEAN DEFAULT false );

Table 3 — Paid Clients

Copy SQL -- Your active client database — created from checkout on payment confirmation CREATE TABLE paid_clients ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, created_at TIMESTAMPTZ DEFAULT now(), checkout_id UUID REFERENCES checkout_submissions(id), first_name TEXT, last_name TEXT, email TEXT, phone TEXT, business_name TEXT, industry TEXT, tier_name TEXT, amount_paid INTEGER, stripe_payment_id TEXT, build_status TEXT DEFAULT 'new', -- new | in_progress | delivered | complete delivery_deadline TIMESTAMPTZ, delivered_at TIMESTAMPTZ, notes TEXT, follow_up_sent BOOLEAN DEFAULT false );

Table 4 — Page Events

Copy SQL -- Lightweight analytics: page views + CTA clicks CREATE TABLE page_events ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, created_at TIMESTAMPTZ DEFAULT now(), event_type TEXT, -- 'page_view' | 'cta_click' | 'audit_start' | 'report_shown' page TEXT, detail TEXT, session_id TEXT );

Enable Row Level Security (run this too)

Copy SQL -- Lock down tables so only your app can write, only you can read ALTER TABLE audit_leads ENABLE ROW LEVEL SECURITY; ALTER TABLE checkout_submissions ENABLE ROW LEVEL SECURITY; ALTER TABLE paid_clients ENABLE ROW LEVEL SECURITY; ALTER TABLE page_events ENABLE ROW LEVEL SECURITY; -- Allow anon inserts (so your site can write data in) CREATE POLICY "Allow anon insert" ON audit_leads FOR INSERT TO anon WITH CHECK (true); CREATE POLICY "Allow anon insert" ON checkout_submissions FOR INSERT TO anon WITH CHECK (true); CREATE POLICY "Allow anon insert" ON paid_clients FOR INSERT TO anon WITH CHECK (true); CREATE POLICY "Allow anon insert" ON page_events FOR INSERT TO anon WITH CHECK (true);
After running all 5 SQL blocks: Go to Table Editor in the left sidebar. You should see all 4 tables listed. If they're there — you're set.
Step 03

Get your Supabase keys

These are the two pieces of information your website uses to communicate with your database. You'll need both.

1

Go to Project Settings

Inside your Supabase project → click the gear icon (Settings) in the left sidebar → click API

2

Copy your Project URL

Looks like: https://xyzabcdef.supabase.co
Copy this and save it.

3

Copy your anon public key

It's a long string starting with eyJ... — this is your public API key. Safe to use in your HTML files.

⚠️ Never share or use your service_role key in your HTML files. Only the anon key goes in client-side code.
4

Find the two lines below in each HTML file

Every file has a section at the top of the <script> that says YOUR_SUPABASE_URL and YOUR_SUPABASE_ANON_KEY — replace those two placeholders with your real values.

Step 05

Email alerts — get notified instantly

Set this up so you get an email the second someone pays or completes an audit. Free through Supabase + Resend.

1

Create a Resend account

Go to resend.com — free plan sends 3,000 emails/month. Sign up and create an API key. Save it.

2

Create a Supabase Edge Function

In Supabase → Edge Functions → New Function → name it notify-owner. This runs automatically when a new paid client row is inserted.

3

Paste this function code

import { serve } from 'https://deno.land/std/http/server.ts' serve(async (req) => { const { record } = await req.json() await fetch('https://api.resend.com/emails', { method: 'POST', headers: { 'Authorization': `Bearer YOUR_RESEND_KEY`, 'Content-Type': 'application/json' }, body: JSON.stringify({ from: 'SystemForge AI <[email protected]>', to: '[email protected]', subject: `💰 New Client: ${record.business_name} — $${record.amount_paid/100}`, html: `<h2>New paid client!</h2> <p><strong>${record.first_name} ${record.last_name}</strong></p> <p>${record.business_name} · ${record.industry}</p> <p>Package: ${record.tier_name} — $${record.amount_paid/100}</p> <p>Email: ${record.email} · Phone: ${record.phone}</p>` }) }) return new Response('OK') })
Replace YOUR_RESEND_KEY with your Resend API key and YOUR_EMAIL with your Gmail address.
4

Create the database webhook

In Supabase → Database → Webhooks → Create webhook. Set it to fire on INSERT on the paid_clients table. Point it to your Edge Function URL. Done — you'll get an email every time money comes in.

Step 06

Backups — protect everything

Supabase free plan includes daily backups retained for 7 days. Here's how to make sure you have an extra layer.

1

Weekly manual export

Every Sunday: Supabase → Table Editor → select any table → click the download icon → Export as CSV. Save to a folder called SF_Backups in Google Drive. Takes 2 minutes.

2

Upgrade to Supabase Pro when ready

At $25/month, Pro gives you Point-in-Time Recovery — meaning you can restore your database to any minute in the past. Once you're generating consistent revenue, upgrade. It's insurance.

Free plan limits to know: 500MB storage (holds ~500,000 leads easily), 2GB bandwidth/month, 50,000 monthly active users. You won't hit these limits in your first year at normal growth.
